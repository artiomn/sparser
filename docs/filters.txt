Список источников -> список условий -> список получателей.

Общий алгоритм вывода:
   1. Произвести чтение изо всех источников.
   2. Произвести разбор выражения.
   3. Найти фильтры, подходящие для данного типа выражения и заданного в фильтре условия.
   4. Проверить входит ли источник в список источников найденных фильтров.
   5. Если источник входит в список фильтров, произвести вывод в приёмник, иначе отбросить сообщение.

Состав точки ввода-вывода:
   1. Тип (сеть/файл/поток).
   2. Параметры типа:
      1. Сеть:
         а. Адрес/порт.
         б. Сокет.
      2. Файл:
         а. Имя.
         б. Дескриптор.
      3. Поток:
         а. Тип.
         б. Дескриптор.
   3. Хэш. Вычисляется по типу и его параметрам.

Состав фильтра:
   1. Тип сообщения. Обязателен.
   2. Источники.  Список точек ввода-вывода.
   3. Прёмники.   Список точек ввода-вывода.
   4. Формат вывода.
   5. Флаги.

Алгоритм создания фильтра:
   1. Получить секцию filter.
   2. Создать структуру фильтра.
   3. Произвести итерацию:
      1. Если элемент тип сообщения, сохранить, как тип. Установить флаг получения типа.
      2. Если элемент - выражение, сохранить.
      3. Если элемент - поле from:
         1. Заполнить структуру точки ввода-вывода.
         2. Найти точку ввода-вывода в списке источников.
         3. Если не найдена, продублировать структуру в список источников фильтра.
         4. Добавить структуру в список фильтра [может повторяться два раза].
      4. Если элемент - поле to:
         1. Заполнить структуру точки ввода-вывода.
         2. Найти точку ввода-вывода в списке приёмников.
         3. Если не найдена, продублировать структуру в список приёмников фильтра.
         4. Добавить структуру в список фильтра [может повторяться несколько раз].
   4. Если флаг получения типа не установлен, разрушить фильтр и вернуть ошибку.
   5. Вернуть код корректного завершения.

Алгоритм вывода:
   1. Создать базу фильтров и сепараторов. Добавить в базу источник и назначение по умолчанию.
   2. Прочитать данные из источника в базе.
   3. Определить тип сообщения из заголовка.
   4. По типу найти следующий фильтр для этого сообщения.
   5. Если фильтр найден, источник фильтра совпадает или отсутствует,
      и выражение совпадает или отсутствует, вывести сообщение в приёмники фильтра
      или, если приёмник фильтра отсутствует, в приёмник по умолчанию.
   6. Если ещё есть фильтры, перейти на пункт 4.
   7. Если фильтры не были найдены, произвести вывод в приёмник по умолчанию.
   8. Если остались сообщения, перейти на п. 2.
